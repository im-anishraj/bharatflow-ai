# DPDP-Aware Workflow Usage

BharatFlow AI does not add a legal compliance system in the MVP. It provides workflow patterns that help teams design privacy-aware customer journeys.

## MVP Guidance

- Tell users why data is collected before asking for it.
- Collect only the minimum data needed for the workflow.
- Avoid collecting sensitive documents unless the workflow owner has a clear purpose and retention plan.
- Use explicit consent steps for sales follow-up, WhatsApp outreach, document collection, and marketing communication.
- Route privacy or deletion requests to a human-owned process.
- Do not send personal data to AI providers unless the workflow owner has approved that processing.

## Template Pattern

Every India-market template should include:

- Purpose statement.
- Consent question when follow-up or document processing is involved.
- Clear handoff route for complex or sensitive requests.
- Result fields that make ownership and follow-up obvious.

## V1 Product Gap

Before external launch, add first-class support for:

- Consent event history.
- Data processing purpose.
- Audit logs.
- Retention policy controls.
- PII redaction.
- Exportable compliance reports.
