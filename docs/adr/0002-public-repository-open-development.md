# ADR 0002: Public Repository Open Development

## Status

Accepted.

## Context

BharatFlow AI is derived from Typebot and remains governed by the Functional Source License in `LICENSE`.

The repository can be public for transparent development, community contribution, review, education, and prototype collaboration. Public source visibility is separate from commercial hosted availability.

## Decision

Make the repository public and operate it as an open-development prototype with professional contribution, security, CI, review, and community management practices.

The project will continue to avoid commercial hosted launch, resale, or externally hosted customer access until a compliant legal path is documented.

## Consequences

- Keep `LICENSE`, `NOTICE.md`, and original copyright notices intact.
- Keep branch protection, CI, CodeQL, Dependabot, and secret-scanning posture active where GitHub supports it.
- Welcome public issues, discussions, and pull requests under the contribution rules.
- Use GitHub private security advisories or maintainer email for vulnerabilities.
- Keep Typebot trademarks out of new BharatFlow customer-facing surfaces except legal provenance references.
- Treat India-focused templates, docs, and landing polish as high-value public contribution areas.
